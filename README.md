# Peptide Sequence to SMILES Generator

A Python-based tool for converting **peptide sequences with chemical modifications** into **SMILES representations** using RDKit.

This project supports natural amino acids, modified residues, non-natural building blocks, and peptide cyclization, enabling flexible molecular construction from sequence-like inputs.

---

## Overview

This tool parses a modified peptide sequence in MAP Format and constructs the corresponding SMILES for that peptide. It resolves residues and modifications from the provided JSON database and assembles the final molecule using RDKit.

The repository also includes a lightweight HTML interface for browsing supported amino acids and modifications with ease.

A Python package version is also available on PyPI:

```bash
pip install pepmodsmi
```
---

## Features

* Conversion of peptide sequences to SMILES
* Support for:

  * Standard amino acids
  * Post-translational modifications (PTMs)
  * Terminal modifications (N- and C-terminus)
  * Non-natural residues
  * D-amino acids (stereochemistry inversion)
  * Peptide cyclization (head-to-tail and residue pairing)
* Batch processing support
* JSON-based extensible modification database

## Usage

### Single Sequence

python stitch.py "ACDA{d}GHIOW"


---

### Batch Processing

python stitch.py -i input.txt -o output.csv


**Supported input formats:**

* `.txt`       #One peptide sequence per line in MAP Format.
* `.fasta`

**Output:**

sequence, SMILES (In case of txt file)

ID, sequence, SMILES (In case of pasta file)


**Errors** are written to:

output.csv.errors

## Sequence Syntax

The tool uses a structured MAP format for representing modifications:

### Basic Sequence
ACDEFG


### Post-translational Modification
A{ptm:mod}C


### Terminal Modifications
DRP{nt:mod}

ACD{ct:mod}


### Non-natural Residues
AG{nnr:CODE}KPW


### D-amino Acids
A{d}CYQS


### Cyclization

**Head-to-tail:**
ACDE{cyc:n-c}

**Disulfide bonds:**
ACDC{cyc:2-4}

## How It Works

1. **Parsing**
   The input sequence is tokenized into residues and modification instructions.

2. **Database Lookup**
   Residues and modifications are resolved using the JSON database.

3. **Molecule Construction**
   Each residue is converted into an RDKit molecule with defined connection points.

4. **Stitching**
   Residues are joined via peptide bonds to form the backbone.

5. **Cyclization**
   Optional ring closures are introduced.

6. **Stereochemistry Handling**
   D-amino acids are generated by inverting chiral centers.
